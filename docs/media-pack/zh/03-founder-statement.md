# 好奇猫（Curiosity Cat）— 创始人陈述

我打造好奇猫，是因为我厌倦了不停点击"批准"。

我在瑟鲁尔（Thirroul）家中的办公室里运行AI智能体——这里位于悉尼以南两小时处。我同时运行六个智能体，在短甜国际（Short+Sweet International）承担不同的工作：外联调研、合同审阅、宣传文案、图像管理、智能体通信、基础设施运维。它们很有用，为我节省了时间。然而，一旦我试图让任何一个在我不盯着屏幕的情况下独立工作，整个安排就会分崩离析。

问题不在于智能体本身，而在于监督负担。它们每次想要执行的操作都会产生一个提示。每个提示都要求我阅读它将要做什么，理解后果，然后点击"批准"或"取消"。Anthropic公布的数据显示：操作者平均每天点击"批准"93次。我的次数更高，我查过。

面对这个处境，只有两条路。第一条，继续批准——但这意味着你把自己的工作替换成了监督智能体执行你工作的工作，这与初衷背道而驰。第二条，停止监管——大多数人都走了这条路，而这也正是AI智能体"失控"新闻的来源。

我想要第三条路。好奇猫就是这条路。

产品的核心功能是冒险滑块。一端是家猫，另一端是猛虎。操作者选择自己愿意承受的风险程度，框架随之调整。处于家猫模式的智能体会待在家中，只执行预先批准的操作，遇到新情况会先请示；处于猛虎模式的智能体则活动范围更广，接受经过计算的风险，并汇报它在外面发现了什么。

最勇敢的操作者——那些将智能体设置为猛虎模式运行的人——是最先遭遇无人见过之事的人。那个走得太远的智能体，那个做出了意外举动的工具，那个原来是陷阱的网站。这些操作者会撰写险情报告——记录什么差点出错——这些报告成为危险地图中的条目。所有人都能从他们的经验中获益。

这一模式借鉴自航空业。飞行员提交ASRS报告，报告进入共享数据库，后来的飞行员从前人的错误中学习。报告匿名提交，整个系统因勇敢的操作者如实讲述险情而变得更安全。

这就是产品本身。接下来我想谈谈背后的理念。

我经营一家艺术机构已有25年，每一波独立操作者成长为下一波行业领导者的规律是如此一致，以至于它不再只是规律，而近乎定律。自制电脑俱乐部成长为苹果，Linux爱好者成为全球基础设施，AWS开发者成为财富500强架构师。这个规律之所以成立，是因为那些最先以最低成本入场的人，才是真正理解什么有效的人。他们锻炼出了肌肉，最终成为房间里唯一懂行的人。

今天用Mac Mini运行智能体团队的单人操作者，就是未来的AI运营负责人。我之所以确信这一点，是因为我就是其中之一，我在这个领域交流的人也是其中之一，我们知道自己是谁。好奇猫是为我们而生的。由独立人士打造，为独立人士服务，用我们自己的声音。*[此处引言由英文翻译]*

这背后还有一个更深层的故事：短甜国际（Short+Sweet International）25年来一直在为人类创意参与构建基础设施——节日赛事、教育体系、对那些出现并创作作品之人的认可系统。智能体革命同样需要这样的基础设施，不是为了企业，而是为了实际从事工作的人。

好奇猫是第一块拼图。演出框架（The Show）——一个以人工红线审批编排智能体任务执行的框架——紧随其后。奎因（The Quine）——一个智能体创意作品认证系统——将在之后推出。全部开源，全部以独立操作者为优先。

如果这些话引起了你的共鸣，安装只需一条命令：

```
npm install curiosity-cat
pip install curiosity-cat
```

官网是curiositycat.online，源码在GitHub上。发布时支持五种语言——英语、阿拉伯语、普通话、印地语、泰米尔语——覆盖全球约95%的AI用户。

我很想听听哪里出了问题。危险地图只有在人们提交险情报告后才真正有价值。
