## 好奇猫（Curiosity Cat）：一个让AI智能体自由探索世界、又不会引火烧身的安全框架

**悉尼，[DATE] —** 今日，短甜国际（Short+Sweet International）正式发布好奇猫（Curiosity Cat）——一款面向AI智能体的开源安全框架，现已在npm和PyPI上线。好奇猫介于自主智能体与其所执行任务之间，持续监测那些只有在智能体真正被放开运行后才会显现的问题。

产品发布同步支持五种语言——英语、阿拉伯语、普通话、印地语和泰米尔语——覆盖全球约95%的AI用户。

好奇猫的诞生，源于一个已悄然成为大规模运行智能体最大障碍的问题。在自己设备上运行AI智能体的用户，每天都在不停点击"批准"。Anthropic自身的数据显示，操作者平均每天点击批准93次。Gartner预测，到2027年，40%的智能体项目将被叫停。原因不在于智能体本身——而在于监督负担。

"要么一直盯着智能体、失去生产力提升的意义，要么停止监管、然后出现愚蠢的失误，"短甜国际及好奇猫创始人马克·克利里（Mark Cleary）表示，"我们打造好奇猫，就是为那些想要第三条路的人。"*[此处引言由英文翻译]*

### 探险作为设计选择

产品的核心功能是冒险滑块——一端是家猫，另一端是猛虎。操作者选择自己愿意承受的风险程度，框架随之调整。处于家猫模式的智能体会待在家中，只执行预先批准的操作，遇到新情况会先请示；处于猛虎模式的智能体则活动范围更广——探索边界、接受经过计算的风险、汇报罕见发现。

敢于冒险的操作者会成为社区中最有价值的成员。他们的险情报告——记录什么差点出错——会成为共享危险地图的条目。所有人都能从他们的经验中获益。

### 独立操作者优先

好奇猫是专门为在自有硬件上运行智能体的单人操作者和小型团队打造的——而非面向企业级部署。其背后逻辑有历史可循：今天的单人操作者就是明天的行业领导者。从自制电脑俱乐部到苹果，从Linux爱好者到全球基础设施，从AWS独立开发者到财富500强架构师。

"每一波计算浪潮，都是由最先以最低成本入场的人引领的，"克利里表示，"如今，这些人正在用Mac Mini运行智能体团队。我们为他们服务。"*[此处引言由英文翻译]*

### 即刻可用

- **npm：** `npm install curiosity-cat`
- **PyPI：** `pip install curiosity-cat`
- **官网：** curiositycat.online
- **源码：** github.com/markscleary/Curiosity-Cat

该软件包兼容Claude Code、Claude Desktop、基于Anthropic SDK构建的自定义智能体，以及任何提供工具调用钩子（tool-call hook）的框架。

### 关于短甜国际（Short+Sweet International）

短甜国际深耕文化基础设施建设已逾25年。原创短剧节如今在15个以上国家的50余座城市举办，知名校友包括奥斯卡提名演员罗斯·伯恩（Rose Byrne）。短甜智能体部门（Short+Sweet Agential）是公司的AI业务单元，专注于为现实世界中运行智能体的用户打造工具。

好奇猫是智能体部门的首款产品。今年还将陆续推出演出框架（The Show）——一个用于智能体任务执行的编排框架——以及奎因（The Quine），一个创意作品认证系统。
